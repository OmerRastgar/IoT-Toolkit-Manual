# On-Premise IoT Docker Stack

This folder contains a pre-configured Docker stack for IoT development, including an MQTT Broker (Mosquitto), a Flow Engine (Node-RED), and a Time-Series Database (InfluxDB).

## Services Included

1.  **Mosquitto (MQTT)**: Port `1883` (Default) and `9001` (WebSockets).
2.  **Node-RED**: Port `1880`. Used for automation and dashboards.
3.  **InfluxDB**: Port `8086`. Used for data storage and visualization.

## Quick Start

### 1. Prerequisites
- [Docker Desktop](https://www.docker.com/products/docker-desktop/) installed and running.

### 2. Launch the Stack
Open your terminal in this folder and run:
```bash
docker-compose up -d
```

### 3. Accessing services
- **Node-RED**: [http://localhost:1880](http://localhost:1880)
- **InfluxDB**: [http://localhost:8086](http://localhost:8086)
    - **Username**: `admin`
    - **Password**: `iotpassword123`
    - **Org**: `iot-toolkit`
    - **Bucket**: `sensor_data`

## ESP32 Connection (Arduino Sketch)

Use the following code to connect your ESP32 to this local MQTT broker.

### Required Libraries
- **WiFi.h** (Built-in)
- **PubSubClient** (Install via Arduino Library Manager)

### Sample Code
```cpp
#include <WiFi.h>
#include <PubSubClient.h>

// Replace with your network credentials
const char* ssid = "YOUR_WIFI_SSID";
const char* password = "YOUR_WIFI_PASSWORD";

// REPLACE with your Computer's Local IP Address (e.g., 192.168.1.50)
// To find it, run 'ipconfig' (Windows) or 'ifconfig' (Mac/Linux) in terminal
const char* mqtt_server = "192.168.x.x"; 

WiFiClient espClient;
PubSubClient client(espClient);

void setup_wifi() {
  delay(10);
  Serial.println();
  Serial.print("Connecting to ");
  Serial.println(ssid);

  WiFi.begin(ssid, password);

  while (WiFi.status() != WL_CONNECTED) {
    delay(500);
    Serial.print(".");
  }

  Serial.println("");
  Serial.println("WiFi connected");
  Serial.println("IP address: ");
  Serial.println(WiFi.localIP());
}

void callback(char* topic, byte* message, unsigned int length) {
  Serial.print("Message arrived on topic: ");
  Serial.print(topic);
  Serial.print(". Message: ");
  String messageTemp;
  
  for (int i = 0; i < length; i++) {
    Serial.print((char)message[i]);
    messageTemp += (char)message[i];
  }
  Serial.println();
}

void reconnect() {
  while (!client.connected()) {
    Serial.print("Attempting MQTT connection...");
    if (client.connect("ESP32Client")) {
      Serial.println("connected");
      client.subscribe("esp32/output");
    } else {
      Serial.print("failed, rc=");
      Serial.print(client.state());
      Serial.println(" try again in 5 seconds");
      delay(5000);
    }
  }
}

void setup() {
  Serial.begin(115200);
  setup_wifi();
  client.setServer(mqtt_server, 1883);
  client.setCallback(callback);
}

void loop() {
  if (!client.connected()) {
    reconnect();
  }
  client.loop();

  // Send a test message every 5 seconds
  static unsigned long lastMsg = 0;
  if (millis() - lastMsg > 5000) {
    lastMsg = millis();
    char msg[50];
    snprintf(msg, 50, "Temperature: %d", random(20, 30));
    Serial.print("Publishing message: ");
    Serial.println(msg);
    client.publish("esp32/temperature", msg);
  }
}
```

## Deployment Methods

### Method A: Localhost / Local Network (No TLS)
- **Best for**: Student labs, home automation units.
- **Port**: `1883`
- **Security**: None (Anonymous allowed).
- **ESP32 Code**: Use the "Sample Code" provided above.

### Method B: VPS / Cloud Deployment (Secure TLS)
- **Best for**: Projects accessible over the internet.
- **Port**: `8883`
- **Security**: SSL/TLS Encrypted.

#### 1. Generate Certificates
Run the helper script in the `onprem-docker` folder:
```powershell
.\generate_certs.ps1
```
Follow the prompts to enter your VPS IP. This will create certificates in `mosquitto/certs`.

#### 2. Enable SSL in Mosquitto
Edit `mosquitto/config/mosquitto.conf` and uncomment these lines:
```conf
cafile /mosquitto/certs/ca.crt
certfile /mosquitto/certs/server.crt
keyfile /mosquitto/certs/server.key
```

#### 3. Secure ESP32 Code (Method B)
Use `WiFiClientSecure` instead of `WiFiClient`.

```cpp
#include <WiFi.h>
#include <WiFiClientSecure.h>
#include <PubSubClient.h>

const char* ssid = "YOUR_WIFI_SSID";
const char* password = "YOUR_WIFI_PASSWORD";
const char* mqtt_server = "YOUR_VPS_IP"; // e.g. 123.456.78.90

// The CA Certificate generated by the script (ca.crt)
const char* ca_cert = \
"-----BEGIN CERTIFICATE-----\n" \
"YOUR_CA_CERTIFICATE_CONTENT_HERE\n" \
"-----END CERTIFICATE-----\n";

WiFiClientSecure espClient;
PubSubClient client(espClient);

void setup() {
  Serial.begin(115200);
  WiFi.begin(ssid, password);
  while (WiFi.status() != WL_CONNECTED) delay(500);

  // Set the CA certificate for verification
  espClient.setCACert(ca_cert);

  client.setServer(mqtt_server, 8883); // Secure port
  // ... rest of the code is same as standard MQTT sample ...
}
```

### Method C: HTTP Server (REST API)
Node-RED acts as a web server to receive data via standard HTTP POST requests.

- **Port**: `1880`
- **Node-RED Setup**: Use an `http in` node. Set Method to `POST` and URL to `/data`.
- **ESP32 Code**:
```cpp
#include <WiFi.h>
#include <HTTPClient.h>

void loop() {
  if (WiFi.status() == WL_CONNECTED) {
    HTTPClient http;
    http.begin("http://192.168.x.x:1880/data");
    http.addHeader("Content-Type", "application/json");
    
    int httpResponseCode = http.POST("{\"temperature\":25.5}");
    Serial.println(httpResponseCode);
    http.end();
  }
  delay(10000);
}
```

### Method D: CoAP Server (UDP)
CoAP is a specialized web transfer protocol for use with constrained nodes and constrained networks.

- **Port**: `5683` (UDP)
- **Node-RED Setup**: 
    1. Go to **Manage Palette** -> **Install**.
    2. Search and install `node-red-contrib-coap`.
    3. Use a `coap in` node listening on port `5683`.
- **ESP32 Code**:
```cpp
#include <WiFi.h>
#include <WiFiUdp.h>
#include <coap-simple.h>

WiFiUDP udp;
Coap coap(udp);

void setup() {
  coap.start();
}

void loop() {
  String message = "{\"temp\":24}";
  coap.put(IPAddress(192, 168, x, x), 5683, "sensor", message.c_str());
  delay(5000);
}
```

## 🏗️ Data Integration Hub

![System Architecture](../docs/assets/images/toolkit/architecture.png)

Node-RED serves as the central brain of this stack. It receives data from different protocols and normalizes them before sending them to InfluxDB.

1.  **Input Nodes**: Create `mqtt in`, `http in`, and `coap in` nodes.
2.  **Processing**: Use a `function` node to format all incoming data into a consistent JSON object.
3.  **Storage**: Connect the output to the `influxdb out` node pointing to the `sensor_data` bucket.

---

## Testing the Pipeline

1.  **MQTT**: Open Node-RED, drop an `mqtt in` node with topic `esp32/temperature`, and a `debug` node. You should see the ESP32 readings appear.
2.  **InfluxDB**: Use an `influxdb out` node in Node-RED to save the readings to the `sensor_data` bucket.
3.  **Visualization**: Use InfluxDB's built-in "Data Explorer" to visualize your time-series data.

---

## Troubleshooting
- **Firewall**: Ensure your computer's firewall allows incoming connections on port `1883`.
- **IP Address**: Make sure the ESP32 is on the same network as your computer and uses the computer's **Internal IP** (not `localhost`).
